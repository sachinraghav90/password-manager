# Phase 7: Dashboard Experience

The Dashboard is the landing page upon logging in. It shifts the paradigm from "a list of data" to "actionable intelligence".

## Layout & Modules

1. **Top Header:** 
   * Greeting: "Good morning, Alex."
   * Global Search Bar (prominent).
   * Primary Action: Large "+ New Item" button.

2. **Security Health Overview (Hero Section):**
   * A visual gauge or score (e.g., "85% Security Score").
   * Quick stats: 3 weak passwords, 1 reused password, 0 compromised.
   * Call to action: "Improve your score" -> links to Watchtower/Security screen.

3. **Recently Accessed (Carousel or Grid):**
   * The 4-5 items the user used most recently.
   * Purpose: Drastically speeds up workflow for daily tasks (e.g., Github, AWS, Email).

4. **Favorites (List):**
   * Items explicitly marked as favorites.

5. **Vault Summaries (Bottom Section):**
   * Cards showing "Personal Vault (142 items)", "Engineering Team (45 items)".
   * Click to dive directly into that vault's list view.

## Why this works?
Instead of dumping the user into an alphabetical list of 500 passwords, the dashboard surfaces the 1% of data they need 99% of the time (Recents/Favorites) and keeps security hygiene top-of-mind.
